## High-Level System Shape

- `sensai-backend` is the system of record.
- `sensai-frontend` is the main user-facing app.
- Persistence is primarily SQLite with hand-written SQL and soft-delete timestamps.
- AI behavior is concentrated in backend route modules and prompt files.
- The frontend mixes:
  - direct backend calls via `NEXT_PUBLIC_BACKEND_URL`
  - server-side backend calls via `BACKEND_URL`
  - Next.js proxy route handlers for NextAuth, Judge0, and Notion OAuth
  - a separate assessment-v3 flow currently hardcoded to `http://localhost:8001`

## Major Domain Areas

- Organizations / schools
- Users and roles
- Cohorts and batches
- Courses, milestones, tasks
- Learning materials, quizzes, assignments
- Chat history and AI tutoring/evaluation
- Scorecards and analytics
- Notion integration
- Assessment generation engines
- Code draft persistence and Judge0 execution proxying

## Key Observations

- The backend is modular at the router and DB-helper level, but still relies heavily on broad Pydantic model files and large route modules.
- The frontend is feature-rich, but several pages and components are very large and serve as orchestration hubs rather than thin composition layers.
- The assessment-v3 frontend currently bypasses environment-driven backend URLs and directly calls `http://localhost:8001`.
- The backend scheduler currently handles scheduled task publishing and a memory check, while BigQuery sync helpers exist but are not visibly scheduled from `scheduler.py`.
